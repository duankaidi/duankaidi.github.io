前置知识：线段树合并，可持久化线段树，边分治，可能会用到一点点虚树。

P4565

边分树神题啊。。。

题意

给定两棵边有边权的树 T1,T2T_1, T_2T1​,T2​，结点数都为 nnn。设 di(x)d_i(x)di​(x) 表示第 iii 棵树上 xxx 的带权深度，
求一组点对 (x,y)(x, y)(x,y)，使得 d1(x)+d1(y)−d1(p)−d2(p′)d_1(x) + d_1(y) - d_1(p) - d_2(p')d1​(x)+d1​(y)−d1​(p)−d2​(p′) 最大，
其中 p,p′p, p'p,p′ 分别代表 x,yx, yx,y 在两树上的 LCALCALCA。为了方便，只需输出最大值。

题解

先考虑变换一下这个式子，设 disi(x,y)dis_i(x, y)disi​(x,y) 表示 (x,y)(x, y)(x,y) 在第 iii 棵树上的距离。那么原式就等于 12(dis1(x,y)+d1(x)+d1(y)−2d2(p′)){1 \over 2}(dis_1(x, y) + d_1(x) + d_1(y) - 2d_2(p'))21​(dis1​(x,y)+d1​(x)+d1​(y)−2d2​(p′))。
于是沿用通道那题的做法，在 T1T_1T1​ 上新建结点 i′i'i′ 与 iii 连接，边权为 d1(i)d_1(i)d1​(i)，接着在 T2T_2T2​ 上枚举 p′p'p′，找出最长路就行了？！
很不幸，这道题的边权可以是负数。所以无法合并最长路。

于是换一种思路，考虑将 dis1(x,y)dis_1(x, y)dis1​(x,y) 转化一下。将 T1T_1T1​ 边分治，在一次分治过程中，设 h(x)h(x)h(x) 表示 xxx 到分治中心一侧的距离，答案变成了 p(x)+p(y)+d1(x)+d1(y)−2d2(p′)p(x) + p(y) + d_1(x) + d_1(y) - 2d_2(p')p(x)+p(y)+d1​(x)+d1​(y)−2d2​(p′)，设 valx=px+d1(x)val_x = p_x + d_1(x)valx​=px​+d1​(x)，那么又变为 valx+valy−2d2(p′)val_x + val_y - 2d_2(p')valx​+valy​−2d2​(p′)。在 T2T_2T2​ 中建出当前分治块的虚树，然后枚举一下 p′p'p′ 就行了。复杂度 O(nlog⁡2n)\mathcal{O}(n \log^2 n)O(nlog2n)。
然而，注意到 n⩽366666n \leqslant 366666n⩽366666，所以这种做法虽然简单，但如果实现不好就需要大量卡常。而事实上，我们也能给出更为优秀的做法。

考虑先枚举 p′p'p′，然后算它的子树在 T1T_1T1​ 中对答案的贡献。于是不得不提到一个科技，叫边分树。

边分树是由 nnn 个长度为 O(log⁡n)\mathcal{O}(\log n)O(logn) 的链合并而成的。而且是一棵二叉树。

想要构建出 nnn 个链，一种方法就是：初始时建立 nnn 个二叉链，每个都为空，称第 iii 个节点对应第 iii 条链。对树进行边分治，设 u,vu, vu,v 为分治边 eee 的两端点，钦定 dep(u)<dep(v)dep(u) < dep(v)dep(u)<dep(v)，将 uuu 那一侧的所有点对应的二叉链底端的左儿子赋为 eee，将 vvv 侧的所有点对应的二叉链底端的右儿子赋为 eee。然后进行下一次分治。

也就是说一条二叉链上的点都是原树的一条边。

当两个边分树合并时，其方法类似于线段树合并，这里就不细说。最后合并完 nnn 个二叉链的边分树等同于边分治时边与边的关系树，并且也是一棵二叉树。

边分树最强的地方就是可以维护某些信息。回到这道题，先枚举 p′p'p′，这时递归它在 T2T_2T2​ 的儿子，每个儿子都维护一个它们的点集合并出来的边分树。在合并两个儿子的时候，可以顺便在合并时统计答案。思考 (x,y)(x, y)(x,y) 在边分治时应该何时统计它们的答案，应该是 x,yx, yx,y 在同一分治块时。对应到边分树上就是：到某一深度前，x,yx, yx,y 的边分树左右儿子关系都相同，在这一深度时，左右儿子关系出现第一次不同。那么为了统计答案，边分树上每个节点维护：这个点代表连通块内 valvalval 的最大值。合并时，当前两个节点祖先左右儿子关系一定相同（类似线段树合并），答案不难统计。时间复杂度 O(nlog⁡n)\mathcal{O}(n \log n)O(nlogn)。

CF757G

边分树练习题

题意

给定一棵 nnn 个结点的树和一个长度为 nnn 的排列 ppp，边有边权。接下来有 mmm 个操作，每个操作有两种类型：

  • 1 l r x 询问 ∑i=lrdis(pi,x)\sum\limits_{i = l}^r dis(p_i, x)i=l∑r​dis(pi​,x)。
  • 2 x，交换 px,px+1p_x, p_{x + 1}px​,px+1​，满足 x<nx < nx<n。

强制在线。

题解

看到路径问题，可以想到点分治。于是一个初步的做法就是，记 kpi=ik_{p_i} = ikpi​​=i，建立一棵点分树，记 disxdis_xdisx​ 为 xxx 到分治中心的距离，点分树上每个点维护分治块内以 kik_iki​ 为下标的，disidis_idisi​ 为值的动态开店权值线段树。但是这样时空复杂度都是 O(nlog⁡2n)\mathcal{O}(n \log^2 n)O(nlog2n)，这道题有点卡空间，不是很推荐。

考虑边分树，沿用暴力写挂的思路，可以研发出可持久化边分树。类似于主席树，先边分治一遍弄出每个点对应的二叉链，然后 rootiroot_irooti​ 在 rooti−1root_{i - 1}rooti−1​ 的基础上合并上 pip_ipi​ 对应的二叉链即可。这里边分树每个结点维护的信息显然是分治块内 disdisdis（到分治边一侧的距离）之和。

这样在询问的时候，对于第一种操作，找到 xxx 对应的二叉链，然后进行一个 queryqueryquery，用 rootrroot_rrootr​ 的答案减去 rootl−1root_{l - 1}rootl−1​ 的答案就行了。对于第二种操作，可以发现，竟然这只会改变 rootxroot_xrootx​ 这一棵可持久化边分树！所以找到 px+1p_{x + 1}px+1​ 对应的二叉链，然后暴力重构一下 rootxroot_xrootx​ 就行了。
